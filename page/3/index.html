Vue生命周期

简单的说生命周期就是事物从产生到消失的一个时间过程，那么vue的生命周期就是其从创建到销毁的过程，其中包含了开始创建、初始化数据、编译模板、挂载dom（渲染），渲染->更新->渲染、销毁（卸载）等一系列过程。
其中钩子函数都担负着自己的职责，较为常用的就是created和mounted函数，可以在函数中编写相应的业务逻辑。

钩子函数

beforeCreate

在实例初始化之后，数据观测和事件、生命周期初始化配置之前被调用。

created

实例已经创建完成之后被调用。在这一步，实例已经完成以下的配置：数据观测、属性和方法的运算，事件回调。然后挂载阶段还没开始，$el属性目前不可见。

beforeMount

在挂载之前被调用：相关的render函数首次被调用，此时有了虚拟DOM

mounted

el被创建的vm.$el替换，并挂载到实例上去之后调用该钩子，渲染为真实DOM

beforeUpdate

在数据更新之前时调用，发生在虚拟 DOM 重新渲染和打补丁之前。 你可以在这个钩子中进一步地更改状态，这不会触发附加的重渲染过程。

updated

由于数据更改导致的虚拟 DOM 重新渲染和打补丁，在这之后会调用该钩子。

当这个钩子被调用时，组件 DOM 已经更新，所以你现在可以执行依赖于 DOM 的操作。然而在大多数情况下，你应该避免在此期间更改状态，因为这可能会导致更新无限循环。

值得注意的是：该钩子在服务器渲染期间不被调用

beforeDestroy

实例销毁之前调用。此时实例仍然是可用的

destroyed

vue 实例销毁后调用。调用后，vue 实例指示的所有东西都会解绑，所有的事件监听器会被卸载移除，所有的子实例也会被销毁。

值得注意的是：该钩子在服务器端渲染期间不被调用。

基于Nodejs的主流桌面应用开发框架NW.js和Electron介绍

NW.js和Electron都可以用前端的知识来开发桌面应用。
NW.js和Electron可以用Nodejs中几乎所有的模块。NW.js和Electron不仅可以把html写的web页面打包成跨平台可以安装到电脑上的软件，也可通过JavaScript访问操作系统原生的UI和Api

Electron把html、css、JavaScript组成的程序构建为跨平台桌面应用程序的原理

通过将Chromium和Node.js合并到同一个运行时环境中，并将其打包为Mac、Windows和Linux系统下的应用来实现这一目的

准备

  1. node.js
  2. git

创建

  1. 安装electron npm install -g electron
  2. 创建项目 通过git克隆项目

  • 克隆实例项目的仓库：git clone https://github.com/electron/electron-quick-start
  • 进入这个仓库： cd electron-quick-start
  • 安装依赖并运行 npm install && npm start
  1. 通过electron-forge 创建项目
    electron-forge相当于electron的一个脚手架，可以让我们更方便的创建、运行、打包electron项目
  • 全局安装electron-forge npm install -g electron-forge
  • 创建项目 electron-forge init my-new-app
    注意：这一步会默认安装模块，如果失败删掉node_modules，重新cd到项目里面运行 cnpm install
  • cd 项目
  • 运行项目 npm start
  1. 通过electron手动创建项目
  • 新建一个文件夹（文件夹不能是中文）
  • 新建一个index.html和一个main.js
  • npm init生成一个package.json

电脑连上网却上不了网问题

发表于 2019-12-06 | 分类于 技术

连着两次用迅雷下电影断网，开机重启并没有奏效，尝试了很多种办法，最后找到简单的实现办法：

  1. 首先通过cmd打开命令框
  2. 然后输入ipconfig/release释放IP地址
  3. 接着利用ipconfig/flushdns来重建本地DNS缓存
    这样，再去网页就可以上网了
