一、变量

var：可重复定义，不能限制修改，没有块级作用域。

ES6引入let、const：不能重复定义；let是变量，const是常量；

解构赋值：

  1. 左右两边必须一样，右边是合法的内容
  2. 赋值和定义必须同步完成。

二、函数

箭头函数

==箭头函数，内外this不变==

  1. 如果有且仅有1个参数，()可以省略。
  2. 如果函数体只有一句话，且是return，则{}可以省略。

默认参数

(a, b = XXX, c = XX)

参数展开（剩余参数、可变参数）

  1. “…”的第一个作用：结构剩余参数
    (a, b, …args): 用args接收剩下的所有参数
  2. “…”的第二个作用：用来展开一个数组

三、数组和json

数组新增

map、reduce、filter、forEach、Array

map

映射：一一对应
[1, 2, 3, 4] => [false, true, false, true]

reduce(汇总)

一堆 -> 一个

filter(过滤)
forEach(遍历)
Array

Array.from([array-like]) => {x, x, x}

json变化：

  1. 简写：名字和值一样的，可以省略

  2. function可以不写

四、字符串

字符串模板

startsWith

判断是否以…开始

endsWith

判断是否以…结尾

五、面向对象

箭头this、bind
统一标准化：class、constroctor；extend、super。写法上的变化。

bind: 给一个函数定死一个this

普通函数：根据调用的人，this经常变
箭头函数：根据所在环境，this恒定

编译：一个语言编程另一语言。
打包：多个文件打成压缩包，减少HTTP请求，提高性能。

六、Promise

异步操作同步化。类似语法糖，将异步请求看上去同步化
作用：解决异步操作。

同步：串行，简单、方便
异步：并发，性能高、体验好

==用同步的方式写异步：==

Promise.all()

  1. Promise有用，解除异步操作
  2. Promise有局限：带逻辑的异步操作较麻烦

Promise.all(), 与：所有的都成功。
Promise.race(), 或：只要有一个完成。

七、generator生成器

生成器函数：能暂停，通过==yield==暂停

yield: 可以传参和返回值

generator和promise配合

generator和promise配合需要一个外来runner辅助执行，会带来不统一、不标准、性能低。且generator函数不能写成箭头函数。
