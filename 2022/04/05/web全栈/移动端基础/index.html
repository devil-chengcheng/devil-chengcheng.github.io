移动端适配

  1. viewport：可视区
    最早手机要照顾PC端

  2. 盒模型box-sizing
    普通盒子.width = width + padding + border
    盒模型:盒子总的大小不变，不再向外部扩展，仅内部

  3. flex-弹性盒模型

    • 具备border-box的能力
    • 对border、padding和margin都好用
    • 跟max-width、min-width配合使用

  4. rem

    • em：相对单位，相对自身字体大小，配合字体大小使用
    • rem：相对单位，相对root字体大小
      相对于html的font-zize。
      总结：在不同的屏幕尺寸下，只需调整HTML元素的font-size，方便、性能高

touch

所有DOM3事件都应该绑定
touch单点事件：

方向锁定：
跟按下的位置进行比较，如果横向移动超过5px，锁定为水平；如果纵向移动超过5px，锁定为垂直。
阶段一：方向确定之前不动
阶段二：方向已经确定了再动

多点：
touches：整个屏幕上的手指
targetTouches：当前物体上的手指数组（不同物体之上的手指不会互相干扰）

不需要做多点触摸的时候选取平均位置
x = 所有手指x / n
y = 所有手指y / n

手势识别：

  • 旋转：旋转后的角度 - 旋转前的角度
  • 缩放：缩放后的距离 / 缩放前的距离

多点触摸：

  1. 避免影响——消除干扰：平均坐标
  2. 需要多点——手势：旋转、缩放
